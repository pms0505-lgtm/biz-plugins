# 업무별 프롬프트 템플릿

사업관리본부 업무에 바로 쓸 수 있는 Claude Code 프롬프트 템플릿 모음입니다.

---

## 기본 구조 (모든 역할 공통)

```
[역할/배경]
나는 사업관리본부에서 {담당 업무}를 맡고 있어.
{현재 상황이나 필요한 배경 1~2줄}.

[요청]
{구체적으로 해줬으면 하는 것}.

[형식]
{원하는 출력 형식: 표/목록/문단/슬라이드 구조 등}.
{길이 제한: A4 1페이지, 3~5줄, 5분 발표 분량 등}.
```

---

## 문서 작성 템플릿

### 보고서 초안

```
나는 사업관리본부에서 {담당 업무}를 담당하고 있어.
{보고서가 필요한 배경}.

{주제}에 대한 보고서를 작성해줘.
- 대상 독자: {팀장/부문장/경영진/외부}
- 핵심 메시지: {전달하고 싶은 핵심 1가지}
- 분량: {A4 N페이지 / 슬라이드 N장}
- 포함할 내용: {현황/분석/제안/결론}
```

### 기획서 구조 잡기

```
{프로젝트명} 기획서를 작성하려고 해.
목적: {왜 이걸 하는지}
대상: {누구를 위한 건지}
범위: {무엇을 할 건지}

먼저 기획서 목차 구조를 3~5개 섹션으로 잡아줘.
각 섹션에 들어갈 내용을 한 줄씩 설명해줘.
목차가 OK면 섹션별로 내용을 써줄게.
```

### 경영진 보고 자료 요약

```
다음 내용을 경영진 보고용으로 요약해줘:
{원본 내용 붙여넣기}

형식:
- 결론 먼저 (1~2줄)
- 핵심 수치 3개 (변화 포함)
- 리스크 또는 이슈 (있으면)
- 요청사항 또는 다음 단계

전체 A4 1/2페이지 이내.
```

## 데이터 분석 템플릿

### 데이터 요약 분석

```
다음 데이터를 분석해줘:
{데이터 붙여넣기}

기준:
- 기간: {YYYY년 MM월 ~ MM월}
- 단위: {원/명/건/% 등}
- 비교 기준: {전월 대비/전년 동월 대비/목표 대비}

분석 항목:
1. 전체 요약 (1~2줄)
2. 주요 트렌드 3가지
3. 이상치 또는 특이사항
4. 추천 액션 (선택사항)

결과를 표 + 요약 문장 형태로 정리해줘.
```

### 항목별 비교 분석

```
다음 {N}가지 항목을 비교 분석해줘:
{항목1}: {데이터/설명}
{항목2}: {데이터/설명}

비교 기준: {속도/비용/효과/리스크 등}
결과: 비교표 + 추천 의견 (우리 상황: {조건})
```

### 데이터 검증

```
다음 계산이 맞는지 검토해줘:
{계산식 또는 결과}

데이터 출처: {출처}
검토 포인트:
- 합계가 맞는지
- 비율 계산이 맞는지
- 이상한 수치가 있는지
```

## 소통/커뮤니케이션 템플릿

### 회의록 작성

```
다음 회의 내용을 회의록으로 정리해줘:
{회의 내용 또는 메모}

정보:
- 날짜: {날짜}
- 참석자: {이름 (역할)}
- 주제: {회의 주제}

형식:
1. 회의 목적
2. 주요 논의 사항 (항목별)
3. 결정 사항
4. 다음 할 일 (담당자, 기한)
5. 다음 회의 (있으면)
```

### 공식 이메일 초안

```
다음 상황에 맞는 이메일을 작성해줘:
- 수신: {상대방/조직}
- 발신: 사업관리본부 {이름}
- 목적: {이메일의 목적}
- 핵심 내용: {전달할 내용 요점}
- 톤: {공식적/친근/정중}
- 첨부파일: {있으면 언급}

이메일 형식으로 제목도 함께 작성해줘.
```

### 협조 요청문

```
{담당 부서/팀}에 다음 내용으로 협조를 요청하는 공문 초안을 작성해줘.

요청 배경: {왜 필요한지}
요청 내용: {구체적으로 무엇을}
기한: {언제까지}
담당자: {이름, 연락처}

공식 협조 공문 형태로.
```

## 업무 자동화 템플릿

### 반복 업무 분석

```
매주/매달 반복하는 업무가 있어.

현재 과정:
1. {단계 1}
2. {단계 2}
3. {단계 3}

소요 시간: {얼마나 걸리는지}
사용 도구: {엑셀/이메일/내부 시스템 등}

이 중에서:
1. AI가 대신할 수 있는 단계
2. 자동화 방법 (구체적으로)
3. 예상 시간 절감 효과

를 알려줘.
```

### 프로세스 개선 제안

```
현재 {업무명} 프로세스를 개선하고 싶어.

현재 문제점:
- {문제 1}
- {문제 2}

제약 조건:
- 사용 가능 도구: {도구}
- 예산/시간 제한: {제한}
- 변경 불가 부분: {고정 요소}

이 조건 내에서 개선 방안 3가지를 제안해줘.
각 방안의 장단점과 구현 난이도 포함.
```

### 체크리스트 생성

```
{업무/프로젝트명}을 위한 체크리스트를 만들어줘.

목적: {무엇을 위한 체크리스트인지}
사용 시점: {언제 사용하는지}
사용자: {누가 사용하는지}

완료 기준이 명확한 체크박스 형태로,
카테고리별로 묶어서 정리해줘.
```

## 검증/검토 요청 템플릿

### 결과물 검토

```
내가 작성한 {문서 유형}을 검토해줘:
{내용 붙여넣기}

검토 포인트:
- 논리적 흐름이 자연스러운지
- 빠진 내용이 있는지
- {독자}가 이해하기 쉬운지
- 개선할 수 있는 부분

구체적인 수정 제안과 함께 피드백해줘.
```

### 다른 관점 요청

```
다음 {계획/보고서/제안}에 대해 반대 의견이나 약점을 찾아줘:
{내용}

특히:
- {이해관계자}가 반대할 수 있는 포인트
- 빠진 리스크
- 현실적으로 어려운 부분

이 피드백을 반영해서 어떻게 보완할지도 알려줘.
```
