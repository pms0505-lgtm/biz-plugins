# ax-eval-tip: 맞춤 활용 팁

현재 AX 레벨에 맞는 Claude Code 활용법과 프롬프트 템플릿을 제공합니다.

---

## 실행 순서

### 0단계: 최신 Anthropic 팁 수집

다음 쿼리로 웹서치하여 현재 레벨에 맞는 Claude/Claude Code 최신 활용법을 수집합니다:

```
"site:docs.anthropic.com claude prompt engineering best practices"
"anthropic claude code tips features 2025"
```

검색 결과에서 실용적인 팁 1~2개를 추출하여 메모합니다.
검색 실패 시 이 단계를 건너뜁니다.

### 1단계: 현재 레벨 파악

`~/ax-eval/assessments/` 에서 가장 최근 평가 파일을 읽습니다.

파일이 없으면: "아직 레벨 평가가 없습니다. 먼저 `/ax-eval 체크`를 실행해주세요."

읽어야 할 정보:
- `level` (1~5)
- `role` (역할)
- `scores.요청력`, `scores.검증력`, `scores.활용력`, `scores.판단력`

### 2단계: 가장 약한 축 파악

4축 점수 중 가장 낮은 축을 식별합니다. 이 축에 맞는 팁을 우선 제공합니다.

### 3단계: 역할 + 레벨 조합 콘텐츠 출력

1. 역할이 있으면 아래 **역할별 맞춤 팁** 섹션에서 `{역할} × {레벨}` 조합을 찾아 우선 출력합니다.
2. 역할이 `미선택`이거나 조합이 없으면 아래 **레벨별 공통 팁** 섹션으로 fallback합니다.
3. 두 경우 모두 **약한 축 처방** 섹션에서 `{WEAKEST_AXIS} × {역할}` 조합 팁을 함께 출력합니다.

## 역할별 맞춤 팁

역할 × 레벨 조합이 있을 때 우선 사용. 없으면 아래 공통 팁으로 fallback.

### UA마케터

**⭐ → ⭐⭐**
- **핵심**: 광고 요청 시 타겟 / 목표 KPI / 예산 규모를 세트로 전달
- **Before**: "이번 캠페인 소재 문구 써줘"
- **After**: "25-34 여성 타겟, 5월 가정의달 캠페인. 목표는 앱 설치 전환 CTR 3% 이상. 따뜻하고 감성적인 톤으로 배너 카피 3가지 써줘."
- **이번 주 목표**: 다음 소재 요청부터 타겟+KPI 1줄 추가

**⭐⭐ → ⭐⭐⭐**
- **핵심**: 소재 초안을 받은 후 타겟 관점에서 검토 요청
- **검증 프롬프트**: "이 카피를 [타겟 페르소나]가 봤을 때 어떻게 느낄지 평가해줘. 클릭하고 싶어질까?"
- **이번 주 목표**: AI 소재 초안을 받으면 무조건 타겟 관점 검토 1회 추가

**⭐⭐⭐ → ⭐⭐⭐⭐**
- **핵심**: 소재 A/B 테스트를 AI로 사전 설계
- **분해 프롬프트**: "이번 캠페인에서 테스트할 가설 3가지 잡아줘. 각 가설마다 테스트 소재 방향과 성공 기준 지표도 포함해서."
- **이번 주 목표**: 캠페인 기획 단계부터 AI로 가설 → 소재 → 측정 루프 설계

---

### CRM마케터

**⭐ → ⭐⭐**
- **핵심**: 세그먼트 조건을 AI에게 줄 때 기준 / 기간 / 제외 조건을 명시
- **Before**: "재구매 고객 리스트 뽑는 기준 알려줘"
- **After**: "최근 90일 기준, 2회 이상 구매 + 최근 30일 앱 로그인 있는 고객. 단, 이미 이번 달 발송 대상에 포함된 고객은 제외. 이 조건으로 SQL 초안 써줘."
- **이번 주 목표**: 다음 세그먼트 요청부터 기간+제외조건 명시

**⭐⭐ → ⭐⭐⭐**
- **핵심**: 발송 전 AI가 만든 세그먼트 조건을 반드시 샘플로 교차 확인
- **검증 프롬프트**: "이 쿼리로 나올 고객이 예상 범위(약 N만 명)에 맞는지 확인하는 방법 알려줘. 놓칠 수 있는 엣지 케이스도 체크해줘."
- **이번 주 목표**: 발송 전 AI 교차검증을 표준 단계로 추가

**⭐⭐⭐ → ⭐⭐⭐⭐**
- **핵심**: 캠페인 결과 분석을 AI와 단계별로 진행
- **분해 프롬프트**: "이번 캠페인 결과 데이터야: [데이터]. 열람률 → 클릭률 → 전환율 순으로 단계별 분석하고, 각 단계 드롭 원인 가설을 3가지씩 세워줘."
- **이번 주 목표**: 발송 후 결과 분석도 AI와 단계별로 처리

---

### 디자이너

**⭐ → ⭐⭐**
- **핵심**: 시각적 요청 시 분위기 키워드 + 레퍼런스 방향 + 사용 목적을 함께 전달
- **Before**: "썸네일 만들어줘"
- **After**: "SNS 인스타그램 피드용 썸네일. 미니멀하고 모던한 톤. 텍스트는 최소화, 여백 많이. 20대 여성이 저장하고 싶어하는 느낌. 시안 3가지 방향으로 설명해줘 (실제 이미지 생성 아님, 디렉션만)."
- **이번 주 목표**: 다음 디자인 요청부터 톤+목적+타겟 1줄 추가

**⭐⭐ → ⭐⭐⭐**
- **핵심**: AI 시안 방향을 브랜드 가이드라인 기준으로 검토
- **검증 프롬프트**: "이 디자인 방향이 [브랜드명]의 [핵심 브랜드 속성] 과 얼마나 맞는지 평가해줘. 어긋나는 부분과 조정 방향도."
- **이번 주 목표**: AI 시안 받으면 브랜드 정합성 검토 1회 추가

**⭐⭐⭐ → ⭐⭐⭐⭐**
- **핵심**: 콘셉트 발산은 AI로, 최종 방향 결정은 본인이
- **분해 프롬프트**: "이번 [프로젝트] 디자인 콘셉트 방향을 3가지로 나눠줘. 각각 타겟 감성 / 레퍼런스 스타일 / 피해야 할 것을 포함해서. 내가 최종 방향 고를게."
- **이번 주 목표**: 콘셉트 기획 단계를 AI로 발산 → 본인 큐레이션 루프로

---

### 데이터분석가

**⭐ → ⭐⭐**
- **핵심**: 분석 요청 시 데이터 출처 / 기간 / 지표 정의를 명시
- **Before**: "매출 트렌드 분석해줘"
- **After**: "2024년 1월~3월 월별 매출 데이터야 (단위: 만원): [데이터]. 전월 대비 증감률, 주요 상품군별 기여도, 이상치 파악. 표와 해석 문장으로 정리해줘."
- **이번 주 목표**: 다음 분석 요청부터 기간+단위+지표 정의 명시

**⭐⭐ → ⭐⭐⭐**
- **핵심**: AI 분석 결과를 받으면 가정(assumption)부터 확인
- **검증 프롬프트**: "이 분석에서 네가 가정한 전제 조건이 뭐야? 데이터에 결측치나 이상치가 있으면 어떻게 처리했어?"
- **이번 주 목표**: AI 분석 결과 받으면 가정 검증 1회 필수

**⭐⭐⭐ → ⭐⭐⭐⭐**
- **핵심**: 분석 로직 설계를 AI와 단계별로 검토 후 실행
- **분해 프롬프트**: "이 분석을 하기 전에 분석 설계를 먼저 검토해줘. 가설 → 필요 데이터 → 분석 방법 → 결과 해석 기준 순서로. 설계가 맞으면 실행할게."
- **이번 주 목표**: 분석 실행 전 설계 검토를 AI와 1회 진행

### 개발자

**⭐ → ⭐⭐**
- **핵심**: 코드 요청 시 언어 / 프레임워크 버전 / 제약 조건을 명시
- **Before**: "파일 읽는 함수 만들어줘"
- **After**: "Python 3.11, pathlib 사용. CSV 파일 경로를 받아 DataFrame으로 반환하는 함수. 파일 없으면 FileNotFoundError, 인코딩은 UTF-8 기본에 cp949 fallback. type hints 포함."
- **이번 주 목표**: 다음 코드 요청부터 언어+제약조건 명시

**⭐⭐ → ⭐⭐⭐**
- **핵심**: AI 코드를 받으면 엣지 케이스 / 에러 처리 기준으로 리뷰 요청
- **검증 프롬프트**: "이 코드에서 입력이 null이거나 빈 배열일 때 어떻게 동작해? 에러 처리가 충분한지, 빠진 엣지 케이스가 있으면 알려줘."
- **이번 주 목표**: AI 코드 받으면 엣지 케이스 검토 1회 추가

**⭐⭐⭐ → ⭐⭐⭐⭐**
- **핵심**: 구현 전 설계를 AI로 검토하고 확인 후 실행
- **설계 검토 프롬프트**: "이 기능을 [방식]으로 구현하려고 해. 이 접근의 trade-off와 더 나은 대안이 있으면 알려줘. 특히 확장성과 유지보수 관점에서."
- **이번 주 목표**: 1일 1회 구현 전 AI 설계 리뷰 루틴

## 약한 축 처방 (역할별)

`WEAKEST_AXIS`와 역할 조합으로 아래 처방 선택. 역할이 없으면 공통 처방 사용.

### 요청력 낮을 때

| 역할 | 처방 |
|------|------|
| UA마케터 | 타겟 페르소나(나이/성별/관심사)를 요청 첫 줄에 항상 포함 |
| CRM마케터 | 세그먼트 조건을 기준/기간/제외 3가지 세트로 전달하는 습관 |
| 디자이너 | 톤앤매너 키워드 3개 + 참고 스타일 1개를 요청에 포함 |
| 데이터분석가 | 분석 목적(의사결정에 어떻게 쓸지)을 요청 앞에 1줄 추가 |
| 개발자 | 언어/버전/제약조건을 첫 줄에 명시하는 코드 요청 템플릿 만들기 |
| 공통 | 요청 전 30초: "AI가 이 요청을 이해하려면 뭐가 더 필요할까?" 생각하기 |

### 검증력 낮을 때

| 역할 | 처방 |
|------|------|
| UA마케터 | 소재 받으면 "타겟이 이 카피에 공감할까?" 타겟 관점 검토 1회 추가 |
| CRM마케터 | 발송 전 AI 세그먼트를 샘플 10~100건으로 직접 확인하는 단계 추가 |
| 디자이너 | 시안 받으면 "브랜드 가이드와 맞는가?" 기준 1가지로 검토 |
| 데이터분석가 | 분석 결과 받으면 "가정이 뭐야?" 먼저 묻기 |
| 개발자 | 코드 받으면 엣지 케이스(null, 빈값, 경계값) 시나리오 검토 |
| 공통 | AI 결과 받으면 "이 중 틀렸거나 부족한 것 찾아줘" 무조건 1회 |

### 활용력 낮을 때

| 역할 | 처방 |
|------|------|
| UA마케터 | 경쟁사 광고 자료를 파일로 첨부해서 분석 요청해보기 |
| CRM마케터 | 데이터 파일(CSV)을 직접 붙여넣거나 Read로 읽히는 방법 시도 |
| 디자이너 | 레퍼런스 이미지 경로를 직접 읽어달라고 요청해보기 |
| 데이터분석가 | 분석을 여러 단계로 나눠 각 단계 결과를 확인하며 진행 |
| 개발자 | Agent/Task 도구로 복잡한 작업 위임 시도 |
| 공통 | 긴 작업은 "먼저 계획 잡아줘, 그 다음 실행" 으로 분리 |

### 판단력 낮을 때

| 역할 | 처방 |
|------|------|
| UA마케터 | AI 소재 방향이 나오면 "왜 이 방향이 이 타겟에게 효과적이야?" 근거 요청 |
| CRM마케터 | 발송 타이밍 제안 받으면 "다른 타이밍 옵션과 비교해줘" 요청 |
| 디자이너 | AI 시안 방향 중 하나를 고를 때 "각 방향의 리스크가 뭐야?" 먼저 물어보기 |
| 데이터분석가 | 분석 방법 제안 받으면 "대안 접근법과 각각의 한계를 비교해줘" |
| 개발자 | 구현 방법 제안 받으면 "이 접근의 trade-off와 대안을 비교해줘" |
| 공통 | AI 제안에 "왜 이 방법이 최선이야?" 항상 1번 묻기 |

## 레벨별 공통 팁 콘텐츠

### ⭐ 입문 단계

**이번 주 추천 습관:**
AI에게 요청할 때 배경을 한 줄 먼저 설명해보세요.

**기본 프롬프트 구조:**
```
나는 [부서/역할]에서 [업무]를 하고 있어.
[배경 설명을 1~2줄].
[구체적으로 해줬으면 하는 것].
```

**예시 (Before → After):**

❌ "보고서 써줘"

✅ "나는 사업관리본부에서 예산 관리를 담당하고 있어.
3월 예산 집행 현황을 정리해야 해.
항목별 집행률과 이슈 사항을 표로 만들어줘."

**이번 주 목표:** 1가지 업무에 AI 적용해보기

---

### ⭐⭐ 활용 단계

**이번 주 추천 습관:**
AI 결과를 받은 후 "이거 맞아?" 하고 한 번 더 확인해보세요.

**활용 프롬프트 구조:**
```
나는 [부서]에서 [업무]를 담당해.
[배경: 상황, 데이터, 조건].
[요청: 구체적으로 무엇을].
[형식: 표/목록/문단 중 어떤 형태로].
```

**검증 습관 예시:**
```
(AI 결과를 받은 후)
"방금 만들어준 표에서 합계가 맞는지 확인해줘."
"다른 방식으로 표현하면 어떻게 될까?"
"경영진이 보기에 이 보고서가 충분한지 검토해줘."
```

**이번 주 목표:** AI 결과를 최소 1번 검토 후 수정 요청

---

### ⭐⭐⭐ 협업 단계

**이번 주 추천 습관:**
복잡한 업무를 단계로 나눠서 AI에게 맡겨보세요.

**업무 분해 프롬프트:**
```
[업무명]을 단계별로 나눠줘.
그리고 각 단계에서 내가 직접 해야 할 것과 AI가 도울 수 있는 것을 구분해줘.
```
